Mapbox APIs used in this demo

The Flyover Route Visualiser is built entirely on Mapbox GL JS v3 (with the Mapbox Standard / Standard Satellite styles). Below is every Mapbox API it uses, grouped by job, with a one-line note on what it does here and a link to the official docs.

← Back to the flyover

Map setup

Creating the map, authenticating, and adding on-map controls.

new mapboxgl.Map()

Creates the map on a globe projection with antialias and the initial Standard Satellite style.

Map class
mapboxgl.accessToken

Sets the public access token (pk.*) that authenticates every tile, style, and model request.

accessToken
new mapboxgl.NavigationControl()

Adds the zoom + compass control (visualizePitch) via map.addControl().

NavigationControl
attributionControl

Keeps the required Mapbox / OpenStreetMap / Maxar attribution and logo on the map.

AttributionControl

Styles & the Standard style config

Swapping base styles per route and driving the Standard style's runtime configuration.

map.setStyle()

Switches the base map between Standard, Standard Satellite, and Satellite Streets — auto-picked per route type.

setStyle
map.setConfigProperty('basemap', …)

Drives the Standard style: lightPreset (dawn/day/dusk/night), show3dObjects, and every label toggle.

setConfigProperty
Mapbox Standard style

The 3D-aware base style whose lighting, buildings, and trees respond to the config above.

Standard style guide
map.setPaintProperty()

Applies live style edits — trail colour/width, the progressive line-trim-offset, and model paint props.

setPaintProperty

3D terrain

Real elevation under the route, and sampling it so the camera clears the terrain.

map.setTerrain() + raster-dem source

Enables 3D terrain from the Mapbox Terrain DEM tileset, with an adjustable exaggeration.

setTerrain
mapbox.mapbox-terrain-dem-v1

The raster-DEM tileset that feeds setTerrain the per-pixel ground heights.

Terrain DEM v1
map.queryTerrainElevation()

Sampled every frame at the camera point so the chase camera flies above the terrain instead of clipping through it.

queryTerrainElevation
sky layer

An atmospheric sky layer added for the classic (non-Standard) styles.

sky layer

Cinematic camera

The heart of the flyover: a free camera walked along the route each frame.

map.getFreeCameraOptions() / setFreeCameraOptions()

The free camera API — each frame we set position and lookAtPoint() to chase the route.

FreeCameraOptions
mapboxgl.MercatorCoordinate

fromLngLat(lngLat, altitude) converts a coordinate + height into the mercator position the free camera needs.

MercatorCoordinate
map.fitBounds() + mapboxgl.LngLatBounds

Frames the whole route in an intro shot before playback begins.

fitBounds
map.once('moveend')

Kicks off the flyover once the intro fitBounds framing settles.

moveend event

Route, trail & 3D model layers

The drawn route, its progressive reveal, and the optional glTF avatar riding the head.

map.addSource() / addLayer() — line

A GeoJSON source (lineMetrics:true) with ghost / casing / trail line layers.

line layer
line-trim-offset

Reveals only the travelled portion of the trail as the camera advances — the "drawing" effect.

line-trim-offset
model source + model layer

A declarative type:'model' source/layer renders the optional glTF avatar (boat / duck).

model source
source.setModels()

Called each frame to move the model to the route head and rotate it to face the direction of travel.

setFog
new mapboxgl.Marker()

Marker

style.load event

Error handling and readiness checks that gate layer setup and playback.

isStyleLoaded
Geometry math (route length, sampling points along the line, bearings) uses Turf.js, which is not a Mapbox API. Requires Mapbox GL JS ≥ 3.19 for the model setModels() animation. Full reference: docs.mapbox.com/mapbox-gl-js.
